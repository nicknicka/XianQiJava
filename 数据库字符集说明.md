# 数据库字符集与乱码问题说明

## 问题分析

经过检查，发现数据库中的**数据本身是正确的UTF-8编码**，之前看到的乱码是因为：

### 原因
使用 MySQL 命令行工具查询时，如果没有指定正确的字符集参数，中文会显示为乱码。

### 数据验证
```sql
-- 错误的查询方式（会显示乱码）
mysql -uroot -p123456 Xianqi -e "SELECT remark FROM \`order\`;"

-- 正确的查询方式（正确显示中文）
mysql -uroot -p123456 --default-character-set=utf8mb4 Xianqi -e "SELECT remark FROM \`order\`;"
```

### 实际数据示例
```sql
remark                | HEX值
----------------------|----------------------------------
下午3点后可以交易      | E4B88BE58D8833E782B9E5908EE58FAF...
周末面交              | E591A8E69CABE99DA2E4BAA4
可以测试吗？          | E58FAFE4BBA5E6B58BE8AF95E59097EFBC9F
```

这些都是正确的 UTF-8 编码！

## 解决方案

### 1. 命令行查询时指定字符集
```bash
# 永久解决：在 ~/.my.cnf 中添加
[client]
default-character-set=utf8mb4

# 临时解决：查询时添加参数
mysql --default-character-set=utf8mb4 -uroot -p123456 Xianqi
```

### 2. 应用程序连接配置
确保 JDBC 连接字符串指定了正确的字符集：

```properties
# application.properties
spring.datasource.url=jdbc:mysql://localhost:3306/Xianqi?useUnicode=true&characterEncoding=utf8mb4
```

### 3. 表和字段字符集
数据库使用 `utf8mb4` 字符集，支持完整的 Unicode 字符（包括 Emoji）。

```sql
-- 查看数据库字符集
SHOW VARIABLES LIKE 'character_set%';
-- 结果：character_set_database = utf8mb4

-- 查看表字符集
SHOW CREATE TABLE `order`;
```
